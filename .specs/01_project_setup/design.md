# Design: Project Setup (Spec 01)

> Design document for the SDV Parking Demo monorepo scaffolding.
> Implements requirements defined in `.specs/01_project_setup/requirements.md`.

## 1. Architecture Overview

The parking demo system is organized as a monorepo containing Rust services, Go backend services, Go mock CLI apps, shared Protocol Buffer definitions, generated code, local development infrastructure, and test suites.

## 2. Module Responsibilities

### 2.1 `proto/`

Contains the single source of truth for all gRPC interface definitions. Three files define the shared contract between Rust and Go components. All files use `syntax = "proto3"` and the `sdv.parking.v1` package.

### 2.2 `gen/go/`

Contains Go code generated by `protoc-gen-go` and `protoc-gen-go-grpc`. This directory is checked into version control so that consumers do not need `protoc` installed to build. It has its own `go.mod` with module path `github.com/rhadp/parking-fee-service/gen/go`.

Sub-packages:
- `commonpb/` -- generated from `common.proto`
- `updateservicepb/` -- generated from `update_service.proto`
- `parkingadaptorpb/` -- generated from `parking_adaptor.proto`

### 2.3 `rhivos/`

Rust workspace for all RHIVOS partition services. The workspace `Cargo.toml` at `rhivos/Cargo.toml` lists four member crates. Each crate is a binary crate with a minimal `main.rs`.

Member crates:
- `locking-service` -- ASIL-B door lock service (Safety partition)
- `cloud-gateway-client` -- Vehicle-to-cloud NATS bridge (Safety partition)
- `update-service` -- Adapter lifecycle manager (QM partition)
- `parking-operator-adaptor` -- Parking session manager (QM partition)

### 2.4 `backend/`

Go modules for cloud-deployed backend services.

- `parking-fee-service/` -- Operator discovery and adapter metadata REST API
- `cloud-gateway/` -- Dual-interface gateway (REST + NATS)

Each directory contains its own `go.mod` with a module path under `github.com/rhadp/parking-fee-service/backend/`.

### 2.5 `mock/`

Go modules for mock CLI applications that stand in for Android apps during development.

- `parking-app-cli/` -- Simulates the AAOS PARKING_APP
- `companion-app-cli/` -- Simulates the mobile COMPANION_APP

Each directory contains its own `go.mod` with a module path under `github.com/rhadp/parking-fee-service/mock/`.

### 2.6 `infra/`

Local development infrastructure managed by Docker Compose.

### 2.7 `tests/`

- `tests/setup/` -- Go test module for spec-01 verification (directory structure, build targets, proto compilation)
- `tests/integration/` -- Placeholder for cross-component integration tests (later specs)

## 3. Makefile Targets

| Target | Behavior | Dependencies |
|--------|----------|--------------|
| `build` | Runs `cargo build --workspace` in `rhivos/`, then `go build ./...` for each Go module in `backend/` and `mock/`. | None |
| `test` | Runs `cargo test --workspace` in `rhivos/`, then `go test ./...` for each Go module. | None |
| `lint` | Runs `cargo clippy --workspace -- -D warnings` in `rhivos/`, then `go vet ./...` for each Go module. | None |
| `check` | Runs `build`, `test`, `lint` in sequence. Fails on first error. | `build`, `test`, `lint` |
| `proto` | Runs `protoc` for each `.proto` file, generating Go code into `gen/go/`. Overwrites existing generated files. | `protoc`, `protoc-gen-go`, `protoc-gen-go-grpc` |
| `infra-up` | Runs `docker-compose -f infra/docker-compose.yml up -d`. | Docker/Podman |
| `infra-down` | Runs `docker-compose -f infra/docker-compose.yml down`. | Docker/Podman |
| `clean` | Runs `cargo clean` in `rhivos/`, removes Go build cache artifacts. | None |

The Makefile uses the `COMPOSE` variable to allow overriding the compose command (defaults to `docker-compose`, can be set to `podman-compose`).

```makefile
COMPOSE ?= docker-compose
```

## 4. Docker-Compose Configuration

File: `infra/docker-compose.yml`

```yaml
version: "3.8"

services:
  nats:
    image: nats:latest
    container_name: sdv-nats
    ports:
      - "4222:4222"
    healthcheck:
      test: ["CMD", "nats-server", "--help"]
      interval: 5s
      timeout: 3s
      retries: 3
      start_period: 5s

  kuksa-databroker:
    image: ghcr.io/eclipse-kuksa/kuksa-databroker:main
    container_name: sdv-kuksa-databroker
    ports:
      - "55556:55555"
    healthcheck:
      test: ["CMD", "grpc_health_probe", "-addr=:55555"]
      interval: 5s
      timeout: 3s
      retries: 5
      start_period: 10s
```

Notes:
- The Kuksa Databroker listens on port 55555 inside the container and is mapped to host port 55556.
- Health checks ensure `docker-compose ps` reports meaningful status.
- The NATS health check uses the `nats-server --help` command as a basic liveness probe (exits 0 if the binary is functional). A more robust check could use a NATS client connection test, but this suffices for local development.

## 5. Protocol Buffer Definitions

### 5.1 `proto/common.proto`

```protobuf
syntax = "proto3";

package sdv.parking.v1;

option go_package = "github.com/rhadp/parking-fee-service/gen/go/commonpb";

// AdapterState represents the lifecycle state of a parking operator adapter.
enum AdapterState {
  ADAPTER_STATE_UNKNOWN      = 0;
  ADAPTER_STATE_DOWNLOADING  = 1;
  ADAPTER_STATE_INSTALLING   = 2;
  ADAPTER_STATE_RUNNING      = 3;
  ADAPTER_STATE_STOPPED      = 4;
  ADAPTER_STATE_ERROR        = 5;
  ADAPTER_STATE_OFFLOADING   = 6;
}

// AdapterInfo describes a parking operator adapter.
message AdapterInfo {
  string adapter_id    = 1;  // Unique identifier for the adapter instance
  string image_ref     = 2;  // OCI image reference (e.g., "registry/repo:tag")
  string version       = 3;  // Adapter version string
  AdapterState state   = 4;  // Current lifecycle state
  string operator_name = 5;  // Human-readable parking operator name
  int64 installed_at   = 6;  // Unix timestamp of installation
  int64 last_active_at = 7;  // Unix timestamp of last activity
}

// ErrorDetails provides structured error information in gRPC metadata.
message ErrorDetails {
  string code                  = 1;  // Machine-readable error code (e.g., "ADAPTER_DOWNLOAD_FAILED")
  string message               = 2;  // Human-readable error message
  map<string, string> details  = 3;  // Additional context key-value pairs
  int64 timestamp              = 4;  // Unix timestamp of when the error occurred
}
```

### 5.2 `proto/update_service.proto`

```protobuf
syntax = "proto3";

package sdv.parking.v1;

option go_package = "github.com/rhadp/parking-fee-service/gen/go/updateservicepb";

import "common.proto";

// UpdateService manages the lifecycle of parking operator adapters on the vehicle.
service UpdateService {
  // InstallAdapter downloads and installs a parking operator adapter from the registry.
  rpc InstallAdapter(InstallAdapterRequest) returns (InstallAdapterResponse);

  // WatchAdapterStates opens a server-streaming RPC that emits adapter state change events.
  rpc WatchAdapterStates(WatchAdapterStatesRequest) returns (stream AdapterStateEvent);

  // ListAdapters returns all currently known adapters and their states.
  rpc ListAdapters(ListAdaptersRequest) returns (ListAdaptersResponse);

  // RemoveAdapter stops and removes an installed adapter.
  rpc RemoveAdapter(RemoveAdapterRequest) returns (RemoveAdapterResponse);

  // GetAdapterStatus returns the current state of a specific adapter.
  rpc GetAdapterStatus(GetAdapterStatusRequest) returns (GetAdapterStatusResponse);
}

// InstallAdapterRequest requests installation of an adapter from the registry.
message InstallAdapterRequest {
  string image_ref        = 1;  // OCI image reference to pull
  string checksum_sha256  = 2;  // Expected SHA-256 checksum of the OCI manifest digest
}

// InstallAdapterResponse confirms the installation has been initiated.
message InstallAdapterResponse {
  string job_id      = 1;  // Unique job identifier for tracking the installation
  string adapter_id  = 2;  // Assigned adapter identifier
  AdapterState state = 3;  // Initial state (typically DOWNLOADING)
}

// WatchAdapterStatesRequest configures the state-watching stream.
message WatchAdapterStatesRequest {
  string adapter_id = 1;  // Optional: filter to a specific adapter. Empty means all adapters.
}

// AdapterStateEvent is emitted when an adapter transitions between states.
message AdapterStateEvent {
  string adapter_id     = 1;  // Adapter that changed state
  AdapterState old_state = 2;  // Previous state
  AdapterState new_state = 3;  // New state
  int64 timestamp       = 4;  // Unix timestamp of the transition
  string message        = 5;  // Optional human-readable description
}

// ListAdaptersRequest requests a listing of all known adapters.
message ListAdaptersRequest {}

// ListAdaptersResponse contains all known adapters.
message ListAdaptersResponse {
  repeated AdapterInfo adapters = 1;
}

// RemoveAdapterRequest requests removal of a specific adapter.
message RemoveAdapterRequest {
  string adapter_id = 1;  // Adapter to remove
}

// RemoveAdapterResponse confirms adapter removal.
message RemoveAdapterResponse {
  AdapterState state = 1;  // Final state after removal (typically STOPPED or UNKNOWN)
}

// GetAdapterStatusRequest requests the current status of a specific adapter.
message GetAdapterStatusRequest {
  string adapter_id = 1;  // Adapter to query
}

// GetAdapterStatusResponse returns the current adapter information.
message GetAdapterStatusResponse {
  AdapterInfo adapter = 1;  // Current adapter information
}
```

### 5.3 `proto/parking_adaptor.proto`

```protobuf
syntax = "proto3";

package sdv.parking.v1;

option go_package = "github.com/rhadp/parking-fee-service/gen/go/parkingadaptorpb";

// ParkingAdaptorService is the common gRPC interface that all parking operator adapters implement.
// The PARKING_APP uses this interface to interact with the running adapter.
service ParkingAdaptorService {
  // StartSession manually starts a parking session, overriding autonomous behavior.
  rpc StartSession(StartSessionRequest) returns (StartSessionResponse);

  // StopSession manually stops a parking session, overriding autonomous behavior.
  rpc StopSession(StopSessionRequest) returns (StopSessionResponse);

  // GetStatus returns the current session status and adapter health.
  rpc GetStatus(GetStatusRequest) returns (GetStatusResponse);

  // GetRate returns the current parking rate for the active zone.
  rpc GetRate(GetRateRequest) returns (GetRateResponse);
}

// StartSessionRequest requests manual start of a parking session.
message StartSessionRequest {
  string zone_id    = 1;  // Parking zone identifier
  string vehicle_id = 2;  // Vehicle identifier (VIN)
}

// StartSessionResponse confirms session start.
message StartSessionResponse {
  string session_id   = 1;  // Assigned session identifier
  bool active         = 2;  // Whether the session is now active
  int64 started_at    = 3;  // Unix timestamp of session start
  string operator_name = 4;  // Parking operator name
}

// StopSessionRequest requests manual stop of a parking session.
message StopSessionRequest {
  string session_id = 1;  // Session to stop
}

// StopSessionResponse confirms session stop and provides fee information.
message StopSessionResponse {
  string session_id    = 1;  // Session that was stopped
  bool active          = 2;  // Should be false after successful stop
  int64 stopped_at     = 3;  // Unix timestamp of session stop
  int64 duration_secs  = 4;  // Total parking duration in seconds
  double total_fee     = 5;  // Calculated parking fee
  string currency      = 6;  // Currency code (e.g., "EUR")
}

// GetStatusRequest requests the current adapter and session status.
message GetStatusRequest {}

// GetStatusResponse returns the current status.
message GetStatusResponse {
  bool session_active   = 1;  // Whether a parking session is currently active
  string session_id     = 2;  // Current session identifier (empty if no active session)
  int64 session_start   = 3;  // Unix timestamp of session start (0 if no active session)
  string zone_id        = 4;  // Current parking zone
  string operator_name  = 5;  // Parking operator name
  bool healthy          = 6;  // Whether the adapter is functioning normally
}

// GetRateRequest requests the current parking rate.
message GetRateRequest {
  string zone_id = 1;  // Optional: specific zone to query. Empty means current zone.
}

// GetRateResponse returns parking rate information.
message GetRateResponse {
  double rate_per_hour = 1;  // Parking rate per hour
  string currency      = 2;  // Currency code (e.g., "EUR")
  string zone_id       = 3;  // Zone the rate applies to
  string operator_name = 4;  // Parking operator name
}
```

## 6. Correctness Properties

These properties define invariants the system must maintain. Each is testable.

### CP-01: Proto Compilation Roundtrip

Running `make proto` followed by `go build ./...` in the `gen/go/` module SHALL succeed without errors. The generated `.pb.go` and `_grpc.pb.go` files SHALL be valid Go source that compiles against the declared `google.golang.org/protobuf` and `google.golang.org/grpc` dependencies.

### CP-02: Rust Workspace Consistency

Running `cargo check --workspace` in `rhivos/` SHALL succeed. Every crate listed in the workspace `Cargo.toml` members array SHALL have a corresponding directory with a valid `Cargo.toml` and at least one source file.

### CP-03: Build Target Idempotency

Running `make build` twice in succession SHALL produce the same result. The second invocation SHALL succeed with exit code 0 and may be a no-op if no source files changed.

### CP-04: Infrastructure Lifecycle Symmetry

Running `make infra-up` followed by `make infra-down` SHALL leave no running containers from the compose file. Running `make infra-down` when no containers are running SHALL succeed (idempotent teardown).

### CP-05: Go Workspace Module Resolution

All Go modules listed in `go.work` SHALL resolve each other's imports without network access to module proxies for workspace-local dependencies. Running `go build ./...` from the repository root (using the workspace) SHALL succeed.

### CP-06: Clean Build from Scratch

Running `make clean` followed by `make build` SHALL succeed. The `clean` target SHALL not delete source files, `.proto` files, or generated protobuf code.

### CP-07: Lint Clean State

Running `make lint` on the initial skeleton code SHALL succeed with exit code 0. The skeleton code SHALL pass both `cargo clippy -- -D warnings` and `go vet ./...` without warnings or errors.

## 7. Error Handling

| Scenario | Detection | Response |
|----------|-----------|----------|
| `protoc` not installed | `make proto` checks for binary | Exit with error: "protoc not found. Install Protocol Buffers compiler." |
| `protoc-gen-go` not installed | `make proto` checks for binary | Exit with error: "protoc-gen-go not found. Run: go install google.golang.org/protobuf/cmd/protoc-gen-go@latest" |
| `protoc-gen-go-grpc` not installed | `make proto` checks for binary | Exit with error: "protoc-gen-go-grpc not found. Run: go install google.golang.org/grpc/cmd/protoc-gen-go-grpc@latest" |
| Proto syntax error | `protoc` exits non-zero | `make proto` propagates error and stops |
| Port conflict on infra-up | Docker compose exits non-zero | `make infra-up` propagates error; user must free the port |
| Docker/Podman not available | Compose command not found | `make infra-up` exits with error: "docker-compose not found" |
| Missing Rust toolchain | `cargo` not found | `make build` exits with error |
| Missing Go toolchain | `go` not found | `make build` exits with error |
| Crate missing from workspace | `cargo build` fails | Cargo reports the missing member; user must create the crate |
| Module missing from go.work | `go work sync` fails | Go reports the invalid module path |

## 9. Definition of Done

Spec 01 is complete when all of the following are true:

1. All directories in the repository structure exist and contain the expected files.
2. `make proto` successfully generates Go code from all three `.proto` files.
3. `make build` compiles all Rust crates and Go modules without errors.
4. `make test` runs all unit tests and passes.
5. `make lint` passes with no warnings or errors on skeleton code.
6. `make check` succeeds end-to-end (build + test + lint).
7. `make infra-up` starts NATS and Kuksa Databroker; both services are reachable.
8. `make infra-down` stops all infrastructure containers cleanly.
9. `make clean && make build` succeeds (clean build from scratch).
10. `go work sync` succeeds with all modules listed in `go.work`.
11. All spec-01 tests in `tests/setup/` pass: `cd tests/setup && go test ./... -v`.

## 10. Testing Strategy

### Unit-Level Verification

Spec-01 tests live in `tests/setup/` as a Go test module. These tests verify:

- Directory existence and expected file presence
- Proto compilation (by running `make proto` and checking output files)
- Go and Rust build success (by running `make build`)
- Makefile target availability (by parsing `make -n <target>`)
- Docker-compose file validity (by parsing the YAML)

### Infrastructure Verification

Infrastructure tests verify:

- Container startup via `make infra-up`
- Port reachability (TCP connect to 4222 and 55556)
- Clean shutdown via `make infra-down`
- No orphan containers after shutdown

### Test Execution

```bash
# Run all spec-01 tests
cd tests/setup && go test ./... -v

# Run build verification
make build

# Run full check
make check

# Run infrastructure lifecycle
make infra-up && make infra-down
```
